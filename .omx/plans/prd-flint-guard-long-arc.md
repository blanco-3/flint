# PRD: Flint Guard Long Arc

## Product Thesis

Flint Guard should evolve from a safer-swap console into an incident-native execution safety operating layer for Solana.

## Core Promise

Flint Guard does not just find a route.
It decides whether execution should happen, what should be unwound during panic events, and why.

## Immediate Strategic Goal

Strengthen the “protective decision system” layer rather than adding more visual surfaces or raw risk heuristics.

## Long-Term Direction

1. Incident Pack
2. Decision Report
3. Panic Action Plan
4. Deterministic audit trail
5. Shared safety feed
6. Wallet / bot / treasury / app integrations
7. Protected execution envelope

## Phase Breakdown

### Phase 1 — Codify Decisions
- Introduce first-class incident model
- Generate structured decision reports
- Produce deterministic audit records
- Keep demo and live paths aligned

### Phase 2 — Productize Operations
- Team-shared incident workflows
- Role-based operator views
- Better integration surfaces for wallets and desks
- Safer export/import and policy portability

### Phase 3 — Safety Infrastructure
- Shared safety feed
- App-consumable policy network
- Event-driven response orchestration
- Stronger execution guarantees around protected envelopes

## Iteration 1 Scope

Implement:
- Incident Pack model
- Decision Report generation
- Panic Action Plan summary
- Deterministic audit trail normalization

Do not implement:
- Multi-user backend
- New on-chain protocol
- Automatic bot remediation
- New repo or architecture reset
